Android Edge AI Runtime for Unity

Setup guidance and issue-reporting information for the Unity Asset Store package.

Setup

Basic installation outline

  1. Import the package into your Unity project.
  2. Open the included sample content or add the runtime component to your scene.
  3. Configure the compatible GGUF model path and runtime parameters.
  4. Confirm that the Android target uses the required architecture and project settings.
  5. Build and deploy to a supported Android device.
  6. Review Unity and Android logs if model loading or inference fails.

Refer to the documentation included with the package for the exact fields, file locations, and version-specific instructions.

Environment

Before reporting a problem

Please prepare the following information:

  • Unity version
  • Package version
  • Android device model
  • Android version
  • Selected GGUF model and quantization
  • Relevant Unity Console or Android log output
  • Clear reproduction steps

Known constraints

Hardware-dependent behavior

  • GGUF model files are not included with the package.
  • Inference speed depends strongly on the Android device and model size.
  • Available memory limits the practical model size and context settings.
  • Not every GGUF model or architecture is guaranteed to be compatible.

Contact

Issue reporting

Technical issues can be submitted through the public project repository. Do not include private information, API keys, licensed model files, or other confidential data in a public report.

Support scope

Support covers the package integration and included runtime components. Third-party model behavior, custom project code, device-specific firmware, and external services may require separate investigation.
